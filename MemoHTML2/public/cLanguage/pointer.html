포인터 공부 시작

포인터를 활용하면 참조 관계를 가진다??

C/C++에서 유일하게 존재하는 것이며 다른 언어는 인터프리터 즉 내부에서 포인터 관리를 해주시 떄문에 pointer의 사용성을 못느낀다

pointer 사용 하는 이유 :

장점!!

  1. 변수의 지역성의 단점 해결 >> 사용자에게 최댜한의 자유도 보장, 참조 관계를 가지게 되어 변수의 수정 자유도를 제공 할 수 있음
  2. 연속된 메모리의 참조 >> float , int , char 등 Array,Struct ,Class 와 같은 자료형(자료구조)들은 연속된 메모리를 할당받는데 이런 자료형을 묶음으로 메모리 주소에 직접 계산 할 수 있게 해주는 것이 pointer 해소 해줌
  3. Heap 영역의 참조 Heap영역은 변수의 동적 할당이 일어나는 부분이며 함수의 스택과는 별도로 잡힌 공간입니다 동적 할당이란: 어떤함수에서 생성했던지 프로그래머가 스스로 삭제를 하거나 프로그램이 끝나지않으면 소멸하지 않는 변수들을 할당하는 것을 동적 할당이라고 한다 이 변수들이 할당하는 부분이 Heap라 한다

    Ex) a()라는 함수안 변수를 동적할당을 이용해서 변수를 만든경우 B()라는 함수 안에서 볼 수 있다. 포인터를 활용하여 Heap영역에 대한 주소를 기억하며 다른 함수에서도 Heap영역을 참조 할 수 있도록한다

단점!!

다루기 많이 까다로움 , 포인터를 이용하여 메모리 구역을 정햇는데 사용자의 운영체제들 접근을 통제해버리는 경우가 있음 , 메모리를 직접 사용하기 떄문에 위험성 또한 많이 크다

pointer(포인터) 정의

  1. 포인터는 메모리의 주소값을 저장한다(특정한 자료를 기억하지않고 그 자료가 저장 되어 있는 메모리 주소를 저장한다)
  2. 포인트 역시 변수라는 것이다

사용되는 특수부호 정리

  • & >> 해당 변수의 주소값을 의미한다 >> 포인터는 주소 값을 지정하는 변수라고 했으니 다른 변수 처럼 일반적인 데이터가 아닌 주소값을 지정해주어야한다
  • * >> 선언시 * 사용하면 pointer를 라고 컴퓨터에게 선언해주는 것이며 *를 변수 >> 에 사용하게 되면 변수의 값을 알아낼때 사용한다 >>pointer일시 그냥 사용하게 되면 주소값이 출력되기 때문에

이중 포인터

        
            int a = 0;
            int *ptr;
            int **sPtr;
            ptr = &a;
            sPtr = &ptr;

이와 같이 **을 사용해주는 이유는 초기에 *ptr이 a의 주소값을 참조하고 있기 때문에 **를 선언해주지 않고 sPtr = &ptr을 해주게 되면 주소를 연달아서 두번 참조하는 경우가 생기는데 이런 경우를 방지하기 위해 **를 사용하여 주소를 건너 다른 주소를 참조 할 수 있게끔 하기 위해서 존재한다
