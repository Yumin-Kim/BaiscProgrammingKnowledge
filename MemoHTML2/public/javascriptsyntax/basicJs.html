기초부터 시작하는 자바스크립트

모든 브라우저는 자바스크립트를 해석하고 실행할 수 있는 자바스크립트 엔진을 내장 브라우저 뿐만아니라 Node.js또한 내장되어 있다 브라우저는 HTML, CSS, 자바스크립트를 실행하여 웹 페이지를 화면에 렌더링하는 것이 주된 목적이지만, Node.js는 서버 개발 환경을 제공하는 것이 주된 목적이다. 따라서 브라우저와 Node.js 모두 자바스크립트의 코어인 ECMAScript를 실행할 수 있지만 브라우저와 Node.js에서 ECMAScript 이외에 추가적으로 제공하는 기능은 호환되지 않는다. 브라우저는 HTML을 조작하는 기능들의 집합을 DOM API(WEB API안에 포함됨) 기본 내장 Node.js는 webAPI 지원하지않고 ECMA Script >> node.js에서 window실행 안됨 Node.js는 chrome V8 자바스크립트 엔진이 빌드된 자바스크립트 런타임 환경이다 브라우저 이외에서 사용됨 서버사이드 애플리케이션 개발에 사용되며 이에 대한 모듈 파일시스템 HTTP 등 빌트인API를 제공한다 ex) createSever() 등등 .. Node.js는 REPL(Read Eval Print Loop)읽고 평가하고 출력 루프 >>가상환경으로 코드 실행 가능

브라우저의 핵심기능은 사용자각 참조하고자 하는 웹 페이지를 서버에 요청하고 서버의 응답을 받아 브라우저에 표시하는것이다
브라우저는 서버로부터 HTML,CSS,Javascript,이미지 파일등을 응답받는다
HTML,CSS파일은 렌더링 엔진의 HTML 파서와 CSS 파서 에 의해 parsing되어 DOM,CSSOM트리로 변화되고 렌더 트리로 결합된다
이렇게 생긴 렌터 트리를 기반으로 브라우저는 웹페이지를 표시한다

자바스크립트는 렌더링 엔진이 아닌 자바스크립트엔진이 처리!!!
HTML 파서는 script태그를 만나면 자바스크립트 코드를 실행하기 위해 DOM 생성 프로세스를 중지
중지하고 자바스크립트 엔진으로 제어 권한을 넘긴다
제어 권한을 넘겨 받은 자바스크립트 엔진은 scirpt태그 내의 자바스크립트 코드 또는 script태그의 src attribute(src ="./,,,")에 정의된 스크립트 파일을 로드하고 파싱하여 실행
자바스크립트의 실행이 완료되면 다시 HTML파서로 제어 권한을 넘겨서 브라우저가 중지했던 시점부터 DOM생성을 재개한다

이처럼 브라우저는 동기적으로 HTML,CSS,Javascript를 처리한다.
이것은 Script의 위치에 따라 블로킹이 발생하여 DOM의 생성이 지연될수 있다는 의미이다
따라서 script 태그 위는 중요한 의미를 갖는다
body 요소의 가장 아래에 자바스크립트를 위치하는게 좋음
-HTML 요소들이 스크립트 로딩 지연으로 인해 렌더링에 지장 받는 일이 발생하지 않아 페이지 로딩 시간 단축
-DOM이 완성되지 않은 상태에서 자바스크립트가 DOM을 조작한다면 에러 발생

패널 설명
Elements 로딩된 웹 페이지의 DOM과 CSS를 편집하여 렌더링된 뷰를 확인해 볼 수 있다. 단, 편집한 내용이 저장되지는 않는다. 웹 페이지가 의도된 대로 렌더링되지 않았다면 이 패널을 확인하여 유용한 힌트를 얻을 수 있다.
Console 로딩된 웹 페이지의 에러를 확인하거나 자바스크립트 소스코드에 포함시킨 console.log 메소드의 결과를 확인해 볼 수 있다.
Sources 로딩된 웹 페이지의 자바스크립트 코드를 디버깅할 수 있다.
Network 로딩된 웹 페이지에 관련한 네트워크 요청(request) 정보와 퍼포먼스를 확인할 수 있다.
Application 웹 스토리지, 세션, 쿠키를 확인하고 관리할 수 있다.

C 나 Java 와는 다르게 변수를 선언할 때 데이터 타입르 미리 지정하지 않는다 변수에 할당된 값의 타입에 동적으로 변수의 타입이 결정된다 이를 동적 타이핑이라 하며 자바스크립트가 다른 프로그래밍 언어와 구별되는점이다

  • 원시타입(primitive data type)
    • number
      자바스크립트의 숫자 타입은 정수만을 위한 타입이 없고 모든 수를 실수를 처리한다.

      Infinity : 양의 무한대
      -Infinity : 음의 무한대
      NaN : 산술 연산 불가(not-a-number)

    • string
    • boolean
    • null
      변수가 기억하는 메모리 어드레스의 참조 정보를 제거하는 것을 의미하며 자바스크립트 엔진은 누구도 참조하지 않는 메모리 영역에 대해 가비지 콜렉션을 수행할 것이다.
    • undefined
      즉, 선언은 되었지만 값을 할당하지 않은 변수에 접근하거나 존재하지 않는 객체 프로퍼티에 접근할 경우 undefined가 반환된다.
    • symbol >> ECMAScript 6
  • 객체 타입(Object data type)
    • Object

문과 표현식의 차이
자연어에서 문이 마침표로 끝나는 하나의 완전한 문장이라고하면 표현식은 문을 구성하는요소이다(그자체로 하나의 문이 될수 있다)
표현식은 평가되어 값을 만들지만 그이상의 행위는 할수없다
문은 var,function과 같은 선언 키워드를 사용하여 변수나 함수를 생성하고
if,for,while과 같은 제어문을 생성하여 프로그램의 흐름을 제어한다
표현식을 통해 평가한 값을 통해 실제로 컴퓨터에세 명령을 하여 무언가를 하는것은 문이다 문의 끝에 붙이는 세미클론은 옵션이며 자바스크립트 엔지이 스크립트를 해석할때 예측되는 지점에 자동으로 삽입(ASI automatic semicolon insertion)

타입변환 명시적 타입 변환(Explicit Coercion) 또는 타입 캐스팅(Type Castiong) 동적 탕비 언어인 자바스크립트는 자바스크립트 엔진에 의해 암뭊거으로 타입이 자동으로 변환되기도한다 암묵적 탕비 변환(Implicit Coercion) 또는 타입 강제 변환(Type coercion) var x = 10; var str = x.toSting(); // 숫자를 문자열로 타입 캐스팅 console.log(typeoof str) string 암묵적 타입변환 숫자 타입 x의 값을 바탕으로 새로운 문자열 타입의 값을 생성해 표현식을 평가한다 var x = 10; var str = x + ""; >>이유 +연산자는 피연산자중 문자열 연결 연산자로 동작한다. 자바스크립트 엔진은 표현식을 평가할때 문맺 즉 컨텐스트에 부합하도록 암묵적 타입변환함 console.log(typeof(str) , str) string 10 x의 값이 변경되지 않는다 console.log(x)//10
