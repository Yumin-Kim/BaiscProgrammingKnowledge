C언어 관련 모든용어 및 문법 정리

  • C언어 메모리 구조

    프로그램을 실행시키면 운영체제는 우리가 실행시킨 프로그램을 위해 메모리 공간을 할당해준다

    할당되는 메모리 공간은 크게 스택(stack) , 힙(Heap) , 데이터(Date) 영역으로 나뉨

    • 할당 시기: 프로그램이 실행될 때마다
    • 할당 장소 : 메인 메모리(RAM)
    • 할당 용도 : 프로그램 실행 시 필요한 메모리 공간(지역 변수 , 전역변수 선언을 위해) 할당
                    
                    #include 
                    void fct1(int);
                    void fct2(int);

                    int b = 20 ; // date영역
                    int main() {
                        int i = 100; // 지역변수 i가 스택 영역에 할당
                        fct1(i);
                        fct2(i);
                        return 0;
                    }
                    void fct1(int c)
                    {
                        int d = 30; // 매개변수 c와 지역변수 d가 스택영역에 할당
                    }
                    void fct2(int e)
                    {
                        int f = 40; // 매개변수 e와 지역변수 f가 스택영역에 할당
                    }
    • 데이터 영역 :

      -전역변수 와 static 변수가 할당 되는 영역

      -프로그램의 시작과 동시에 할당되고 , 프로그램이 종료 되어야 메모리에서 소멸

      프로그램 실행시 main 함수가 호출되기전에 데이터 영역에 할당

      >>전역 변수가 프로그램이 종료 될때 까지 존재 하는 이유
    • 스택 영역 :

      -함수 호출 시 생성되는 지역 변수와 매개 변수가 저장되는 영역

      -함수 호출이 완료 되면 사라짐

    • 힙영역(Heap)

      필요에 의해 동적으로 메모리를 할당 할 때 사용

                              
                            int main()
                            {
                                int i = 10;
                                int arr[i];
                                return 0;
                            }

      i 라는 변수가 10이기 때문에 arr이라는 배열의 크기가 10이라는 것을 알 수 있지 않을까? 결과는 아니다. 컴파일을 하는 동안 i가 4바이트의 크기라는 것을 알 수는 있으나, 그 값이 10으로 초기화 되었다는 사실은 무시하고 넘어간다. 값이 10으로 초기화 되었다는 사실은 실행되는 동안, 즉 런타임에 결정된다. 그렇기 때문에 컴파일러는 arr의 크기가 40바이트가 된다는 사실을 알 수 없다.

      사용자의 요구애 맞게 메모리를 할당해 주기 위해서는(메모리 크기를 결정 하고 싶을 때 ) 메모리 동저 할당을 통해 힙영역에 메모리를 할당 해야 한다

      힙영역은 할당 해야 할 메모리의 크기를 프로그램이 실행 되는 동안 결정 해야 하는 경우(런타임때) 유용하게 사용되는 공간이다


  • 동적 메모리 할당

    malloc(memory allocation) 함수 호출하여 동적 메모리 할당을 요청하면 요구하는 크기의 메모리를 할당하고 그 시작 주소를 반환한다

    (int *)malloc(40) malloc이 반환하는 주소는 타입이 없는 주소(void *)이다. 정수들을 저장하기 위해서는 int *로 반환한다(반드시 필요한 건 아니다)

    malloc으로 할당 받은 메모리는 보통 배열처럼 사용한다


  • 변수 // 상수 정리

    변수는 데이터를 저장하고 재활용 할 수 있는 공간이다. 실제로 컴퓨터의 메모리 안에 저장을 해서 데이터를 다룬다 메모리에 변수가 저장이 되는데 컴퓨터에는 메모리를 구분하기 위한 주소가 존재한다

    상수는 변하지 않는 데이터를 담아놓는공간이다. 즉 한번 메모리에 데이터를 담아두면 추후에 그값을 바꿀수 없다

  • 상수 종류

      둘의 차이는 리터럴 상수는 변하지 않는 데이터,심볼릭 상수는 변하지 않는 변수

      마지막으로, 상수는 선언과 초기화를 동시에 해야 한다.

    1. 리터럴 상수 :1, 2, 3, a, b, c와 같은 읽은 그대로의 의미가 있는 상수
    2. 심볼릭 상수 상수를 기호화하여 의미있는 이름으로 지어서 쓰는 상수

      • const를 사용하는 경우

                                        
                                    const int NUM = 100;
                                    const double PI = 3.14;
                                    printf("NUM = %d \n", NUM);   // 100이 출력
                                    printf("PI = %lf \n", PI);   // 3.14출력
        관습적으로 상수의 명을 대문자로 작성 한다

      • #define을 사용하는 경우

                                        
                                    #define PI 3.14
                                    #define NUM 100
                                    
                                    printf("NUM = %d \n", NUM);
                                    printf("PI = %lf \n", PI);
                                    ////////////////////////////////////////
                                    #include 

                                        #define FOR(I,S,E) for(int I=S;I < E ;I++)
                                        #define TALK(A) (printf("%s\n",A))
                                        int main ()
                                        {
                                            int i;
                                            FOR(i,0,10)
                                            {
                                                printf("Hello %d\n",i);
                                            }
                                             
                                            TALK("Hello CLanguage");
                                            
                                            return 0;
                                        }

        위와 같이 함수 나 여러값들을등 여러개의 명령문을 하나의 명령어로 간단하게 묶어 사용 할 수 있는 기능을 한다

        선언 당시 자료형을 사용하지 않는다 상수의 크기가 0은 아니지만 메모리의 자리를 차지 하지 않는다
