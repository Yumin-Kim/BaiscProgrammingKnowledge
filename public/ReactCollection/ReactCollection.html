Sub Category :

Sub Description :

    Front-End : React + Next + Redux + Redux.Saga

    Back-End : Express + Sequelize(Mysql)

    중점: Front-End,Back-end 각각 서버를 보유하고 있다!!

    프로젝트에는 css프레임워크 사용

    Next.js를 사용하면 SSR이나 Router가 어떻게 사용되는지 모르기 때문에 나중에 사용
    순수하게 SSR,code spliting 한번 해보는것이 중요!!

    파일 분리시
    1. pages(실질적인 페이지 링크 거는 파일들)
    2. container(Redux의 dispatch 사용되는 파일들)
    3. components(Redux를 사용하지 않고 부모에게 받아온 데이터를 뷰에 뿌려주는 파일들)
    4. 서버 담당하는 페이지
    5. 웹펙

    1. 원하는 장르 만들고 싶었던 사이트 기획 필요 >> backend(DB 구성) front-end(DB를 통해 어떤 데이터가 응답이 올지 Dummy Data로 구성)
    2. Front단에 파일 구성 redering만 담당하는 Componenet파일 Dispatch가 포함되는 Component 파일 구성
    3. Front 에서 dummy Data로 기본 구성이 완료 되었으면 backend Express로 어떤 Api사용할건지 고민!!
