* 용어 정리 및 설명


1,OS

Operation System의 약자로써 하드웨어를 제어하고 사용자 프로그램을 통해 하드웨어의 자원을 제어를 방지하며 사용자 편의 제공 및 효울적인 자원 관리를 돕는  소프트웨어를 의미한다.

2,폰노이만 구조 시스템

폰 노이만이 고안한 내장 메모리 순차처리방식으로 데이터 메모리 프로그램 메모리가 구분되어 있지 않고 하나의 버스를 가지고 있는 구조

폰노이만 구조는 소프트웨어만 교체하면되기 때문에 범용성이 뛰어나지만 순차적인 명령 수행하여 병목 현상이 발생한다.

3,커널(os 종류 별로 설명)

사전적인 의미로는 “알맹이”, “핵심”입니다.

운영 체제의 핵심적인 역할로  커널의 역할 역시 운영체제의 핵심 역할 담당한다

보안 , 자원 관리 , 추상화의 역할을 한다.

4,프로세스

컴퓨터에서 연속적으로 실행되고 있는 상태를 말하며 종종 스케줄링의 대상이 되는 작업(task)라는 용어와 거의 유사한 의미로 사용된다(프로세스란 실행중인 프로그램).

프로세스와 프로그램을 혼동하는 경우가 있는데 프로그램은 보조 기억 자치에 존재하며 실행되기를 기다리는 명령어 와 정적인 코드 데이터의 묶음이다 .

이 프로그램의 명령어와 정적 데이터가 메모리에 적재되면 프로세스가 된다

5,버스

CPU가 처리한 데이터들은 모니터에 출력되거나 메모리에 저장되어진다 이러한 행위가 이루어지기 위해서 데이터간의 컴포넌트 통신이 가능 해야한다.(버스는 데이터를 통신 할 수 있게 해주는 시스템이다)System Bus와 I/O Bus 등 다양한 종류가 있고 System Bus가 전송 할 수 있는 정보 따라 Address , Data , Control 가 존재한다.

Address: 데이터 전송을 할때 데이터가 메모리 어디에 있는지를 나타내는 위치

Data: 시스템 요소들 사이에 데이터를 전송하는데 사용 되는 선들의 집합

Control : 제어 신호 전달 및 Read 와 Write신호가 전달 된다.

6.Interrupt - H/W S/W별로 설명

사전적인 의미로는 “끼어들다” , “중단시키다” 정도의 의미를 가진 말로 프로그램을 실행하는 도중에 예기치 않은 상황이 발생할 경우 현재 실행중인 작업을 중단하고 발생된 상황을 처리한 후 다시 실행중인 작업으로 복귀하는 것을 말한다

종류는 외부 ,내부 , 소프트웨어 인터럽트가 존재 한다.

내부

소프트웨어 인터럽트

7.장치 드라이버

하드웨어와 운영체제 간의 사이를 연결하는 통로라고 할 수 있다.

여러 다양한 종류의 하드웨어를 제어하기 위한 모든 정보를 특정 응용 프로그램 내부에 내장 시키는 것을 불가능 때문에 운영체제에는 여러 종류의 하드웨어를 응용 프로그램에서 원활히 제어할 수 있도록 해당 하드웨어의 구동 방식 및 특성, 기능등에 대한 정보를 담고 있는 내장 하고있다.

8.ISR(Interrupt Service Routine)

CPU가 하드웨어적으로 주요 레지스터를 저장라고 인터럽트 벡터 테이블에서 해당 ISR의 위치를 찾아 점프하면 그 ISR에서부터는 운영체제 코드가 작동하기 시작한다. IST은 실제 이터럽트 처리 루틴으로 다시 점프하기 전에 준비 사항을 처리하는 코드와 실제 처리 루틴으로 나눌 수 있다.(인터럽트를 검사하도 처리 방법을 결정하여 처리를 실행 한 다음 논리적 인터럽트값을 반환 한다.)

9,IRQ( Interrupt Request )

컴퓨터의 프로세서는 외부 장치의 인터럽트 요청 회선을 통해 각 장치의 상태를 확인 한다.

IRQ는 데이터 제어선의 일종으로 해당 입력이 발생했을 때만 전송되기 때문에 프로세서가 각 장치의 상태를 직접 점검할 필요가 없게 된다.

10.pooling

마이크로 컨트롤러가 입력을 받아들이는 방법 중 하나로 사용자가 명령어를 사용하여 입력 핀의 값을 계속 읽어서 변화를 알아내는 방식

11.Interrupt Vector(Interrupt Vector Table)

발생하였을 때 해야 할 일이 무엇인가를 정해 놓은 것으로 바로 인터럽트 서비스 루틴의 시작 주소이다.

12 . 장치 컨트롤러 / local buffer

컴퓨터 내의 각 하드웨어 장치에는 컨트롤러가 존재 한다. 컨트롤러는 일종의 작 은 CPU로서 컴퓨터 전체에 CPU라는 중앙처리장치가 있듯이 컨트롤러는 각 하드웨어 장치마다 존재하면서 이들을 제어하는 작은 CPU라고 할 수 있다.

장치 컨트롤러에는 장치로부터 들어오고 나가는 데이터를 임시로 저장하기 위한 작은 메모리를 가지고 있다 이를 로컬 버퍼라고 부른다.

13.Single Processor System

CPU하나로 동작하며 , 예전에 대부분의 시스템은 싱글 프로세서를 사용했다. 장치에 따라 특별한 목적을 가진 프로세서가 들어갔다.

14.Multi Processor System

멀티 프로세서 시스템은 이젠 일반적인 컴퓨터 시스템이 되었다. 멀티 프로세서 컴퓨터는 2개 이상의 프로세서를 가지고 있다. 처음에는 서버 컴퓨터에 처음 적용됐는데, 지금은 모바일 기기도 멀티 프로세서 시스템으로 만들어진다. 멀티 프로세서 시스템은 처리량 , 신뢰성이 증가하는 장점을 가지고 있다.

15.NUMA

(Non-Uniform Memory Access, NUMA)는 멀티프로세서 시스템에서 사용되고 있는 컴퓨터 메모리 설계 방법중의 하나로, 메모리에 접근하는 시간이 메모리와 프로세서간의 상대적인 위치에 따라 달라진다. NUMA구조에서 프로세서는 자기의 로컬 메모리에 접근할 때가 원격 메모리에 접근할 때보다 더 빠르다. 원격 메모리는 다른 프로세서에 연결되어 있는 메모리를 말하고 로컬 메모리는 자기 프로세서에 연결되어 있는 메모리를 말한다.

16.Clustered Systems(대칭 비대칭도 설명)

여러 대의 컴퓨터들이 연결되어 하나의 시스템처럼 동작하는 컴퓨터들의 집합을 말한다. 클러스터의 구성 요소들은 일반적으로 고속의 근거리 통신망으로 연결된다.

17.Multiprogramming and Multitasking

멀티 프로그래밍

입출력 작업의 종료 대기할 동안 하나의 프로세서에서 다른 프로그램을 수행할 수 있도록 하는 것을 의미한다.CPU작업과 입출력 작업을 병행하는 것을 의미한다.

멀티 태스킹

테스크란 OS에서 처리하는 작업단위를 의미하는데 그 여러 작업 단위가 빠르게 처리되어 동시에 처리되는것처럼 보이는 것을 의미하며 Concurrent라는 특징을 생각하면 된다.

Task를 OS의 스케줄링에 의해 task를 번갈아가며 수행하는 것을 의미한다.

18.virtual memory

가상 메모리는 메모리를 관리하는 방법의 하나로, 각 프로그램에 실제 메모리 주소가 아닌 가상의 메모리 주소를 주는 방식을 말한다.

이러한 방식은 멀티태스킹 운영 체제에서 흔히 사용되며, 실제 주기억장치(RAM)보다 큰 메모리 영역을 제공하는 방법으로 사용된다.

19.System call

C나 C++과 같은 프로그래밍 언어 에서 제공하지 않는 기능들은 운영체제(커널)의 서비스를 호출하여 사용하는 것을  말한다. 프로세스 제어, 파일 조작 , 장치 관리 , 정보 유지 , 통신등 사용자 모드가 아닌 커널 모드에서 동작한다.

20.Dual-mode Operation System

사용자와 운영체제는 시스템 자원을 공유한다 . 그렇기 때문에 사용자에게 제한을 두지 않은면 사용자가 메모리 내의 주요 운영체제 자원을 망가뜨릴 위험 생기게 된다 . 운영체제는 원활한 작동과 기능을 위해서는 사용자의 시스템 자원 접근을 제한하는 보호 장치가 필히 필요하다.

21.부트로더 

운영체제가 시동되기 이전에 미리 실행되면서 커널이 올바르게 시동되기 위해 필요한 모든 관련 작업을 마무리하고 최종적으로 운영체제를 시동시키기 위한 목적을 가진 프로그램을 말한다.

22. 펌웨어

하드웨어와 소프트웨어의 특성을 모두 갖춘것을 의미하며 새로운 소프트웨어가 등장하면 새로운 논리 회로를 하드웨어에 새로 추가하기 보다는 하드웨어 내부의 제어 부분에 저장공간을 만들어 그곳에 논리 회로의 기능을 보강하거나 대신 할 수있는 프로그램을 넣을 수 있게 하였다.(프로그램의 형태를 갖추고 있으므로 기능적으로는 소프트웨어에 가깝지만 하드웨어에 내부의 위치하며 사용자가 쉽게 바꿀수 없으므로 하드웨어의 특성을 가지고 있다고 할 수 있다.)

같은 종류의 하드웨어라고 해도 펌웨어가 달라지면 기능이나 서능 혹은 사용하는 소프트웨어의 종류가 달라질 수 있다.

대표적인 예로는 BIOS가 있다.(ROM에 저장 되어서 기기에 전원이 인가 할 시 자동으로 실행되는 프로그램이다.)

-포함

포함 시킴으로써 사용자는 다양한 응용프로그램을 통해 사용성의 편이를 제공 및 사용자의 문제를 수월하게 해결 함으로써 포함을 주장합니다.

-미포함

타 응용 프로그램 공급 업체의 경쟁을 막을 수 있으며 사용하지 않는 미리 포함된 응용 프로그램으로 인해 하드웨어 자원이 더 필요한 경우가 발생으로 인해 미포함을 주장합니다.


1.5 커널 모드 와 사용자 모드의 구별은 기본적인 형태의 보호(보안) 로서 어떤 기능을 하는가?

1.11 분산 시스템의 클라이언트 – 서버 시스템과 피어 간 모델의 차이를 설명하라

분산 시스템은 사용자가 시스템 내의 다양한 자원들을 접근할 수 있도록 네트워크로 연결되어 있고 클라이언트 – 서버 시스템는 클라이언트의 요청에 의해 서버가 응답해주는 방식으로 이루어져 서버는 병목으로 작용하고 있고 분산 시스템과는 다르게 피어 간 모델은 클라이언트 와 서버가 서로 구별되지 않고 시스템상의 모든 노드를 통해 요청응답이 가능하며 서비스가 네트워크에 분산된 여러 노드에 의해 제공될 수 있다.
