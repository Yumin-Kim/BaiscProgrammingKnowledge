OS(Operaion System) 운영체제 정의

OS는 제어 프로그램으로써 컴퓨터의 부적절한 사용을 막거나,사용자 프로그램의 실행을 제어!!
제공하는 하드웨어와 사용자를 연결해주는 인터 페이스를 제공해주며 역할은 사용자 관점 , 시스템 관점으로 나뉜다

  • 서비스 제공자로 응용 프로그램,네트워크 서비스 ,계산 프린트등을 제공함
  • 컴퓨터의 관정에서 운영체제는 컴퓨터 <-> 하드웨어간의 연결을 맺어주는 자원할당자로 또다른 시스템 관정으로 하드웨어 장치와 사용자의 장치룰 제어 할 필요성 때문에 제어프로그램이 될 수 있다

  • 컴퓨터 시스템의 구성

    하드웨어(기본적으로 계선에 필요한 자원을 제공) // 운영체제(사용자와 시스템간의 인터페이스 제공) // 응용 프로그램 (사용자의 계산 문제를 시스템의 자원을 통해 해경하기 위해 정의된 프로그램)

    컴퓨터 시스템 동장

    일반적인 현대의 컴퓨터 시스템은 공유 메모리의 접근을 지원하는 공용 버스(common bus)로 연결된 하나 이상의 프로세서(CPU)와 장치 제어기(Device Contorls)들로 구성되어 있다.
    프로세서와 장치 제어기들은 메모리 사이클을 얻기 위해 경쟁하면서 병행 실행 될 수 있다.

    부트스트랩 프로그램

    이 프로그램은 컴퓨터가 구동을 시작하기 위한 초기 프로그램으로써 시스템 초기화,운영체제 로딩의 역할로 일반적으로 펌웨어로 알려짐 ROM or EEPROM에 저장되어 있다

    운영체제 커널을 적재하고온다!!

    여기서 커널은 운영 체제의 핵심이되는 컴퓨터 프로그래밍의 하나로 시스템의 모든것을 완전히 통제 한다

    그럼 펌웨어 와 드라이버의 차이는 펌웨어는 장치의 ROM(롬은 비휘발성 메모리로 반영구적이다 >> ex) BIOS가 있다)에 저장되어 직접적으로 장치를 제어하고 호환문제 해결
    드라이버 경우 하드웨어와 통신할수 있도록 돕고 하드디스크와 같은 보조기억장치에 저장되어 해당 장치간 구동을 돕는다()

    인터럽트가 발생했을 때 프로세서의 레지스터 정보를 변경해야할 필요가 있다면 인터럽트 루틴은 반드시 어딘가에 현재의 상태를 저장했다가 인터럽트가 종료된 후 인터럽트 호출 전의 상태로 복원해야 한다.

    Interrupt란 CPU가 프로그램을 실행 하고 있을때 입출력 하드웨어등의 장치나 또는 예외 상황이 발생하여 처리가 필요할 경우에 CPU에게 알려 처리할 수 있도록 하는것

  • 멀티태스킹, 멀티프로그래밍, 멀티프로세싱, 멀티스레딩, 멀티코어프로그래밍의 차이

    간단한 용어 정리

    코어 란: CPU안에 기초적인 연산을 허리하고 담당하는 부분을 말함
    Multi Core : 하나의 CPU안에 여러개의 코어를 구현해 놓은 아키텍처를 멀티 코어라고 한다
    Program : 저장소에 존재하는 코드의 무치로 실행 상태가 아닌 실행 하기전에 저장소에 저장된 의미
    Processor: H/w중앙 처리 장치를뜻하며 S/w에서는 데이터 처리시스템을 의미하며 컴파일러등이 있다 >> 즉 CPU ,마이크로프로세서를 의미 Process : 저장소에 존재하는 프로그램이 컴퓨터가 실해해서 CPU가 처리할 수 있게 메인 메모리에 올라온 상태
    Thread : 프로세스 안에 있는 작은 실행 단위로 프로세스와 Data 영역 과 Code영역을 공유하고 스레드 각자의 스택과 레지스터를 갖는다

    Thread가 필요한이유 ??
    멀티태스킹중 >> A 프로세스와 B 프로세스가 동일한 대규모 데이터를 필요로 한다면?? >> 단순히 동이란 데이터를 공유하기 위해서 os 힘듬
    프로세스가 생성되는 초기부터 데이터를 공유 했으면 발상으로 개발됨
    또한 프로세스간의 전환 속도보다 스레드간의 전환 속도가 빠름
    하지만 스레드를 사용하면 동일한 데이터를 전역 변수로 사용하기때문에 다양한 문제 발생!!

    Task : 운영체제가 제어하는 프로그램의 기본단위(프로세스보다 확장된것을 의미함)
    PCB(Process Control Block) CPU가 한번에 하나의 프로세스만 실행이 가능한데 이점을 생가한다면 프로그램 실행중 하던 프로그램을 멈추고 다른 프로그램을 사용하면 그 프로그램은 CPU로 넘어가 전 프로그램 즉 프로세스는 PCB에 보관하게된다(PCB는 중요한 정보를 포함하고있다)
    병렬 처리(parallel processing) : 프로그램 명령어를 여러 프로세서에 분산시켜 동시에 수행함으로써 빠른 시간 내에 원하는 답을 구하는 작업을 일컫는다 >> 프로그램이 입출력을 시작하여 끝나기를 기다리는 동안, 다른 계산 수행을 하는 프로그램이 실행될 수 있도록 함 >>> 동시에 많은 계산을 하는 연산의 한 방법이다

    프로세스 상태

    일반적으로 프로세스는 커널이 관리 하며 프로세스 샅애가 PCB에 저장되어 있고 접근하지 못하는 영역에 저장되어 있다. 이때 커널은 스택 영역은 편리하면서도 보호 받는 위치이기 때문에 주로 이용된다

    프로세스의 상태는 생성(Create), 실행(Running) , 준비(Ready) , 대기(Waiting) , 종료(terminated)

    • 생성(create or new) 상태란, 프로세스 자체는 생성되었지만 아직 프로그램이 메모리에 적재되지 않은 상태를 말합니다.
    • 준비(ready) 상태란, 실행된 프로세스가 메모리에 적재되고 CPU를 할당 받기 위해서 준비중인 상태입니다.
    • 실행(running) 상태란, 프로세스가 CPU를 할당받아서 명령어를 실행하고 있는 상태입니다.
    • 대기(waiting or Block) 상태란, 실행상태에 있던 프로세스가 급작스런 이벤트에 의해 실행을 일시적으로 멈춘 상태를 말합니다.

      • 밑은 간단 용어 정리

    • admitted: 생성된 프로세스가 승인을 받아서 실행이 됩니다.
    • scheduler dispatch: 보통 dispatch라고도 말하며, 준비상태에 있던 프로세스가 CPU를 할당받아서 실행되는 것입니다.
    • interrupt: 실행 중이던 프로세스가 특정 이벤트에 의해서 다시 준비상태로 넘어가는 것입니다. 프로세스가 할당된 시간에 모든 명령을 실행하지 못하여 넘어가는 Time out 등이 있습니다.
      (Interrupt는 Hardware의 Interrupt를 말하며 Software는 Trap이라고 부른다.)
    • I/O or event wait: CPU가 프로세스를 실행 중에 있을 때 급작스런 이벤트에 의해서 CPU가 다른 프로세스를 할당하게 될때 현재 실행중인 프로세스는 대기상태로 넘어가는데 이러한 상태전이를 말합니다.
    • exit: 프로세스의 모든 명령이 실행되고 종료되는 상황을 말합니다.

    • 멀티 코어 프로그래밍 : 멀티코어 프로그래밍이란 하나의 작업을 위해 여러개의 CPU 코어를 사용하기 위해 코드를 작성하는 작업
      멀티 코어 사용 기법의 종류로는 멀티 프로세스 프로그래밍돠 멀티 스레드 프로그래밍이 있다
    • 멀티 프로세싱 : 멀티 프로세싱은 한개 이상의 컴퓨터 프로세서들이 협력하여 프로그램들을 처리하는것
      하나의 작업을 하나의 프로세서(CPU)가 처리하는 것이 아니라 다수의 프로세서가 도와줌!!
      만약 하나의 프로세서가 하나의 작업만을 처리한다면 특정 프로세서가 고장이 났을 때 해당 작업은 정지됩니다. 하지만 멀티 프로세싱을 사용한다면 작업은 정지되지 않습니다. 단지 속도가 느려지는 정도의 손해만 발생하겠죠.
    • 멀티 스레딩 : 멀티 스레드 프로그래밍은 하나의 작업을 위해서 하나의 프로세스에서 여러 스레드를 생성하여 여러 CPU코어를 사용하기 위해 코드를 작성하는 작업을 말한다. 스레드는 하나의 프로세스의 힙 메모리 공간과 텍스트 메모리 공간을 공유하고 있기 때문에 서로 변수나 객체의 교환이 가능하다.
      멀티 스레딩은 스레드 수준뿐 아니라 명령어 수준의 병렬 처리에까지 신경을 쓰면서 하나의 코어에 대한 이용성을 증가하는 것에 초점을 두고 있습니다.
      멀티 스레딩의 장점으로는 자원을 공유하여 메모리에 대한 효율성을 가져올 수 있는 것과 이로 인해 경제성 또한 증가하는 것 등이 있습니다
    • 멀티 프로그래밍 : 멀티프로그래밍은 병렬처리의 초보적인 형태로서, 여러 개의 프로그램들이 단일 프로세서 상에서 동시에 실행되는 것이다. 그러나, 그 컴퓨터에는 오직 단한 개의 프로세서만이 존재하기 때문에, 진정한 의미로는 여러 개의 프로그램이 동시에 수행된다고 볼 수는 없다. 다만, 운영체계의 통제하에, 한 프로그램이 일부 수행되고 나서, 또다른 프로그램이 일부 수행되는 식으로 진행되는 것이다. 그러나, 사용자에게는 모든 프로그램이 마치 동시에 수행되는 것처럼 보인다.
    • 멀티 태스킹 : 다중작업(이하 멀티태스킹)은 다수의 작업(혹은 프로세스, 이하 태스크[1])이 중앙 처리 장치(이하 CPU)와 같은 공용자원을 나누어 사용하는 것을 말한다. 엄밀히 말해 한 개의 CPU를 가진 개인용 컴퓨터가 특정 순간에 수행할 수 있는 태스크의 개수는 하나뿐이다. 따라서 멀티태스킹은 스케줄링이라는 방식을 사용하여 컴퓨터 사용자에게 병렬 연산이 이루어지는 것과 같은 환경을 제공한다.
      멀티태스킹을 사용하게 되면 탑재한 CPU의 숫자보다 많은 수의 태스크를 동시에 수행할 수 있게 된다.
      프로세서가 각각의 Task를 자주 번갈아 가면서 처리 하기 때문에 맞이 동시에 움직이는것처럼 보임
      멀티프로그래밍은 프로세서의 자원이 낭비되는 것을 최소화하기 위한 것이며
      멀티테스킹은 일정하게 정해진 시간동안 번갈아가면서 각각의 Task를 처리하는 것입니다.

  • 운영체제 동작 원리

      운영체제는 작업 도중 Error 로 인해 계속해서 점유하는 일과 같이 컴퓨터 성능을 저해하는 행위를 보호 하는 수단이 필요!!

    1. Dual-Mode-Execution

      유저모드

      유저가 접근할 수 있는 영역 제한,자원 침범 방지,우린 코드 작성후 프로세스를 실행하는 행동을취함,거의 모든 어플리케이션코드는 유저모드에서 실행

      커널모드

      모든 자원(드라이버(os와 입출력장치간의 통역>>커널의 입부분으로 동작함),메모리,CPU)

      유저모드 >> 커널모드
      프로세스가 유저모드에서 실해되다가 특별한 요청이 필요할때 system call을 이용하여 커널에 요청 ex)malloc등등
      커널모드 >> 유저모드
      system call의 요청을 받은 커널이 요청에 대한 일을 하고 결과값을 system call을 리턴 값으로 전함

    2. Timer

      무한루프나 자원의 독점을 막는다

      Timer는 특정 시간이 지나면 Interrupt를 발생시키고 운영체제는 그 시간을 감소시키면서 Interrupt를 기다립니다.
      운영체제는 Timer가 끝난 작업을 종료시키고 또한 실행되기 전 Scheduling 작업 전에 Timer를 작동시킵니다.
