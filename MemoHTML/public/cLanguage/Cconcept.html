  • 임베디드 시스템:

    내장된,안에달려 있다는 의미

    어떤 기계안에 제어를 목적으로 하는 목적으로 하는 아주 작은 컴퓨터가 달려있다고 비유

    기계의 다른 부분을 위해 일하는 모든 컴퓨터'

    간단하게 만드는 것을 목표로 하기 때문에 사용자 입맛에 맞게 무언가를 추가하거나 빼는 일이 상대적으로 어렵다.

  1. C언어와 C++의 차이

    • C++컴파일러가 C코드를 컴파일 할수있지만 >> 입출력방식부터 다르며 C언어에 없는 문법을 C++은 가지고 있다
      • C++은 네임스페이스(namespace)개념을 가진다

        C는 식별자를 같이 사용하면 안되지만 C++은 같은 식별자를 가질수 있다

        컴파일러가 같은 식별자를 사용하면 혼동오는 것을 방지하여 네임스페이스를 사용하여 구분

    • 객체를 이용한 출력(C++) VS 함수를 이용한 출력(C언어)
      • C언어 에서는 printf()를 활용하여 출력하지만 C++에서는 std::cout << "hello, world!" << endl; 이런식으로 출력을 한다
      • cout는 함수가 아니라는 것을 알수있고 내부 장의되어 있는 cout 객체에서의 << 연산자 기능은 출력내용을 버퍼로 삽입하는것이다.
      • 이 버퍼로 삽입된 내용이 모니터에 출력!!
      • C 언어에는 없는 클래스 기반의 객체의 개념을 C++에서 사용할 수 있습니다. 이른바 ‘객체지향 프로그래밍을 지원한다’고 합니다.
    • 표준 입출력 라이브러리가 다르다

      C언어 / C++ 입출력 방식 조사하시!!

  2. 각각의 언어는 어떤 상황하는가??

    • 이식성을 갖춘 프로그램을 만들수 있다
      • 함수를 사용하여 개별 프로그래밍이 가능
      • 이식성을 갖춘 프로그램이란?

        기종이 다른 컴푸터에서도 사용할수있는 프로그램을 말합니다

      • 짜야할 코드의 양이 많아지는 대신 *리소스 관리를 섬세하게 해줄 수 있어 최적화가 용이합니다.
      • 리소스 관리란?

        메모리 사용량/CPU에서 작업할 양과 같이 하드웨어적 자원 사용을 관리하는 것을 말합니다.

    • 사용분야
      • C >> 시스템 프로그래밍(OS), 임베디드 프로그래밍
      • C++ >> 고성능 게임 프로그래밍,서버 프로그래밍
    • 출력방식
      • C>> 함수출력
      • C>> 객체출력

  3. 정확한 차이가 뭐지??

    절차 지향인 C언어를 객체지향적으로 프로그래밍 가는하게 만든 것이 C++

    C언어는 절차 지향적으로 순서에 입각해서 프로그래밍을 한다 하지만 컴퓨터의발전 ,소프트워어의 용량도 커집에 따라 복잡도가 깊어지기 시작

    이 복잡도를 해결하기 위해서 모듈화 하려는 노력이 더해져서 객체 지향인 c++탄생

    객체 지향으로 각각 부품(모듈화)시켜 어느 하나의 부분이 오류가 나면 그부분을 찾아서 수정!!
